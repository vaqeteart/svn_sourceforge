1、直接调试：F5
2、设置断点（再直接可多步F5）
3、运行到当前光标处：CTRL+F10
4、设置断点：F9
4、单步调试：F10
5、函数进入:F11 Shift+F11(跳出函数)
6、设置调试步的位置（不是后退）:右键
7、调试的过程当中可以修改后面的变量。


高级技巧：
ctrl+b
中condition中对应行的变量如果变化了，那么就断点。这个意思是，定位到那行，然后如果那行有一个变量l，把l变量名输入这个对话框之后，如果调试中到了这一行，就会提示：这行中的l如果变了，那么在它变得地方停下来。
有的地方不能下断点.比如空行,基本类型定义语句(不初始化),等等非执行语句.比如
int i;  // 此行不能下断点
int j=0;// 这里可以下
CStringstr;// 这里可以下
intk = Max(i,j);// 这里可以下

有时候即使在debug下也不能下断点,在你的代码的任意地方加上
  __asm{
  int 3
};
  用debug编译,然后直接运行程序.当程序执行到上述代码的时候,就会出现一个框,告诉你说一大堆信息,说程序错了,下面有"调试"、"终止"、"忽略"，不用理,点调试就可以进入跟 踪了。这种方法我还没有失 手过:)
3/edit->breakpoints,location 中removeall可以去掉所有的断点;location选中一个断点之后，condition可以设置条件中断。
